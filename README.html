任务

目前，我现在测试的任务其实是非常简单。我希望使用二叉树神经网络来建模一个离散映射过程，更具体来说，是二进制元素的映射关系。

对于模型的输入，我称它为一个Query，记为q，它是一个二进制变量，也就是一个布尔变量的向量。

对于模型的输出，我称它为一个Value，记为v，它也是一个二进制变量，一个布尔变量的向量。

树模型的任务就是建模这样一个确定的映射关系，q->v，对于任意的一个q，它可以学习到需要输出哪一个具体的v。

方法

对数域的运算

假设现在有一个命题A，对于该命题，我们去刻画它为真和为假的信念分别是多少。

P(A)=exex+eyP(¬A)=eyex+eyP(A) = \frac{e^x}{e^x + e^y} \\ P(\lnot A) = \frac{e^y}{e^x + e^y} P(A)=ex+eyex​P(¬A)=ex+eyey​

实际上，0和1之间是一个非常窄的区间，在这个区间内求取梯度进行机器学习往往会导致精度上出现问题。一个好的做法是将所有运算转到对数域进行，这并不复杂。

logP(A)=x−log(ex+ey)logP(¬A)=y−log(ex+ey)logP(A) = x - log(e^x + e^y) \\ logP(\lnot A) = y - log(e^x + e^y) logP(A)=x−log(ex+ey)logP(¬A)=y−log(ex+ey)

举例来说，(x1=−100,y1=0)(x_1=-100, y_1=0)(x1​=−100,y1​=0)和(x2=−10000,y2=0)(x_2=-10000, y_2=0)(x2​=−10000,y2​=0)相比，它们在概率上其实是非常接近，但他们的x却相差了一个两个数量级。因此，通过对数操作，可以直接将x暴露出来，对于精度是有保障的。因此，此后篇幅内所有的运算都是在对数域中进行。

逻辑与

假设现在有一个命题A和命题B，假定这两个命题是独立的，那么命题A∧BA \land BA∧B应该如何刻画呢？

logP(A)=xA−log(exA+eyA)logP(B)=xB−log(exB+eyB)logP(A) = {x_A} - log(e^{x_A} + e^{y_A}) \\ logP(B) = {x_B} - log(e^{x_B} + e^{y_B}) logP(A)=xA​−log(exA​+eyA​)logP(B)=xB​−log(exB​+eyB​)

logP(A∧B)=logP(A)+logP(B)logP(A \land B) = logP(A) + logP(B) logP(A∧B)=logP(A)+logP(B)

逻辑或

logP(A∨B)=logsum(logP(A∧B),logP(¬A∧B),logP(A∧¬B))=logsum(logP(A)+logP(B),logP(¬A)+logP(B),logP(A)+logP(¬B))logP(A \lor B) = logsum(logP(A \land B), logP(\lnot A \land B), logP(A \land \lnot B)) \\ = logsum(logP(A) + logP(B), logP(\lnot A) + logP(B), logP(A) + logP(\lnot B)) logP(A∨B)=logsum(logP(A∧B),logP(¬A∧B),logP(A∧¬B))=logsum(logP(A)+logP(B),logP(¬A)+logP(B),logP(A)+logP(¬B))

logsum也是对数域的一种运算，它的运算方式如下

logsum(x1,x2,⋯ ,xn)=log(∑inexi)=x∗+log(∑inexi−x∗)logsum(x_1, x_2, \cdots, x_n) = log(\sum_i^n e^{x_i}) \\ = x^* + log(\sum_i^n e^{x_i - x^*}) logsum(x1​,x2​,⋯,xn​)=log(i∑n​exi​)=x∗+log(i∑n​exi​−x∗)

其中，x∗=max⁡inxix^* = \max_i^n x_ix∗=maxin​xi​。通过这种拆解技巧，可以有效保持参数在对数域上的精度。另外，logsum运算支持规约操作，在GPU上有非常高效的实现。

logsum(x1,x2,x3,x4)=logsum(logsum(x1,x2),logsum(x3,x4))logsum(x_1, x_2, x_3, x_4) = logsum(logsum(x_1, x_2), logsum(x_3, x_4)) logsum(x1​,x2​,x3​,x4​)=logsum(logsum(x1​,x2​),logsum(x3​,x4​))

非二值命题

可学习命题

来考察一个最简单的一个机器学习问题，模型的输入是一个二值命题A，模型的输出也是一个二值命题B。那么就只会有两种情况B=AB = AB=A 或者 B=¬AB = \lnot AB=¬A。到底是哪一种情况呢？可以设置一个可学习的命题C来进行刻画，

B={Aif¬C¬AifCB = \begin{cases} A \quad if \quad \lnot C \\ \lnot A \quad if \quad C \end{cases} B={Aif¬C¬AifC​

那么B的 logP 就表示为

logP(B)=logsum(logP(A)+logP(¬C),logP(¬A)+logP(C))logP(¬B)=logsum(logP(A)+logP(C),logP(¬A)+logP(¬C))log P(B) = logsum(log P(A) + log P(\lnot C), log P(\lnot A) + log P(C)) \\ log P(\lnot B) = logsum(log P(A) + log P(C), log P(\lnot A) + log P(\lnot C)) logP(B)=logsum(logP(A)+logP(¬C),logP(¬A)+logP(C))logP(¬B)=logsum(logP(A)+logP(C),logP(¬A)+logP(¬C))

此时命题C就可以通过梯度下降来进行学习。通过设置各种各样的可学习命题还可以实现各种各样有趣的功能，比如我们可以对于特征向量学习一个重要性的排序；也可以在排序后特征空间中学习一个边界，当特征向量在边界的左边时，走左子树，而在右边时走右子树。

二叉树

非叶节点

我的二叉树参考了KD树的运行方式，每进入一个节点，会对q中的特征进行重排，取非，边界比较三个步骤，然后会计算出左子树和右子树的对数概率。下面的子树会接收这些对数概率，并根据自己计算的左子树和右子树的对数概率，将接收到的对数概率分裂成两部分再传给自己的下一层。

假设有非叶节点N, N(q)=0N(q)=0N(q)=0表示节点N把q分配到左子树， N(q)=1N(q)=1N(q)=1表示节点N把q分配到右子树。

假设有三个非叶节点N1,N2,N3N_1,N_2,N_3N1​,N2​,N3​，其中N2,N3N_2,N_3N2​,N3​为N1N_1N1​的左右子节点，选择N1N_1N1​的对数概率记为logP(N1)log P(N_1)logP(N1​)。那么有

logP(N2)=logP(N1)+logP(N1(q)=0)logP(N3)=logP(N1)+logP(N1(q)=1)log P(N_2) = log P(N_1) + log P(N_1(q)=0) \\ log P(N_3) = log P(N_1) + log P(N_1(q)=1) logP(N2​)=logP(N1​)+logP(N1​(q)=0)logP(N3​)=logP(N1​)+logP(N1​(q)=1)

沿着二叉树从根节点一路计算下去，就可以得到q相对于每一个叶节点的注意力的对数概率。这里的二叉树，只考虑完全二叉树的情况。

假设树有叶节点N1,N2,N3,⋯NmN_1,N_2,N_3,\cdots N_mN1​,N2​,N3​,⋯Nm​，我们给每一个叶节点分配一个可学习的viv_ivi​，viv_ivi​可以是二值变量也可能是非二值变量。把模型的输出记为Tree(q)Tree(q)Tree(q)，那么

在叶节点这里其实它是根据上层递推过来的注意力来分配或者说检索相对应的v。

分析

为什么二叉树神经网络可以通过梯度下降进行学习？可以做一些简单的分析。

定义一元指数和函数为

f(x;w,b)=∑iewix+biw=[w1,w2,⋯ ,wn]b=[b1,b2,⋯ ,bn]f(x;w, b) = \sum_i e^{w_i x + b_i} \\ w = [w_1, w_2, \cdots, w_n] \\ b = [b_1, b_2, \cdots, b_n] f(x;w,b)=i∑​ewi​x+bi​w=[w1​,w2​,⋯,wn​]b=[b1​,b2​,⋯,bn​]

x是标量，w、b是向量。那么在w、b的空间上就会有一个f的函数族，f是关于x的函数。在本节中，w和b将视为f中的常量，而x会视为自变量。那么显然两个一元指数和函数的和还会是一元指数和函数。

f1(x)=f(x;w1,b1)f2(x)=f(x;w2,b2)f1(x)+f2(x)=f(x;[w1,w2],[b1,b2])f_1(x) = f(x;w^1, b^1) \\ f_2(x) = f(x;w^2, b^2) \\ f_1(x) + f_2(x) = f(x;[w^1, w^2], [b^1, b^2]) f1​(x)=f(x;w1,b1)f2​(x)=f(x;w2,b2)f1​(x)+f2​(x)=f(x;[w1,w2],[b1,b2])

定义一元指数概率函数为

g(x)=f1(x)f1(x)+f2(x)f1(x)=f(x;w1,b1)f2(x)=f(x;w2,b2)g(x) = \frac{f_1(x)}{f_1(x) + f_2(x)} \\ f_1(x) = f(x;w^1, b^1) \\ f_2(x) = f(x;w^2, b^2) g(x)=f1​(x)+f2​(x)f1​(x)​f1​(x)=f(x;w1,b1)f2​(x)=f(x;w2,b2)

显然g(x)一定会是0到1之间的值。对于g(x)的函数族，也有一些显然的性质。如果g1(x)g_1(x)g1​(x)和g2(x)g_2(x)g2​(x)都属于一元指数概率函数这一函数组，那么g1(x)⋅g2(x),1−g1(x),1−g2(x)g_1(x) \cdot g_2(x),1 - g_1(x), 1 - g_2(x)g1​(x)⋅g2​(x),1−g1​(x),1−g2​(x)也都属于该函数族。同理，g1(x)⋅(1−g2(x)),(1−g1(x))⋅g2(x),(1−g1(x))⋅(1−g2(x))g_1(x) \cdot (1 - g_2(x)),(1 - g_1(x)) \cdot g_2(x), (1 - g_1(x)) \cdot (1 - g_2(x))g1​(x)⋅(1−g2​(x)),(1−g1​(x))⋅g2​(x),(1−g1​(x))⋅(1−g2​(x))也都属于该函数族。再者，g1(x)⋅g2(x)+(1−g1(x))⋅(1−g2(x))g_1(x) \cdot g_2(x) + (1 - g_1(x)) \cdot (1 - g_2(x))g1​(x)⋅g2​(x)+(1−g1​(x))⋅(1−g2​(x)) 等g(x)乘积的和也是还是属于这一函数族。因此，有在g(x)的函数族内非常充足的运算来保证，对任意多的g(x)进行乘法和乘积和的操作后，该结果依然属于g(x)的函数族内。

有了一元指数概率函数族这一概念之后，我们会发现，上面介绍的各种二叉树的推理模块，他们都有一个特点，就是保证对数内的函数对于每一个可学习参数而言，永远保持在一元指数概率函数族内。因此，我们只需要对该函数族进行收敛性分析，就可以判断整个二叉树的可收敛的情况。

对于某一个参数x和某一个batch的损失函数而言，其实它的表达式可以是这样：

L=−log(g(x))=−log(f1(x)f1(x)+f2(x))=log(f1(x)+f2(x))−log(f1(x))L = -log(g(x)) = -log(\frac{f_1(x)}{f_1(x) + f_2(x)}) \\ = log(f_1(x) + f_2(x)) - log(f_1(x)) L=−log(g(x))=−log(f1​(x)+f2​(x)f1​(x)​)=log(f1​(x)+f2​(x))−log(f1​(x))

可以对这个式子做一个换元

u=log(f1(x))v=log(f2(x))L=log(eu+ev)−uu = log(f_1(x)) \\ v = log(f_2(x)) \\ L = log(e^u + e^v) - u u=log(f1​(x))v=log(f2​(x))L=log(eu+ev)−u

那么，显然，我们可以知道L是关于u和v的凸函数，也就是在uv平面上，L是一个口朝上的一个大锅。由于u和v都是关于x的增函数，因此我们可以知道，x是在uv平面内的一条单调的曲线。因此我们可以知道，L关于x至少是一个拟凸的函数，因此采用梯度下降求解是有效的。

需要强调的是两点，第一，二叉树中所有的操作都是在保持损失函数对于每一个参数而言都是在 -log(g(x)) 这一函数族内的；第二，只要损失函数对于每一个参数x保持在-log(g(x))函数族内，那么它一定是一个拟凸函数，一定可以在其他参数不变时通过梯度下降求解到使得L最小的值。从而整个模型可以迭代地求出关于训练数据的损失函数最小的模型参数。
